The Rules you write + the Patterns JotFolio finds

Two layers working together. You set the rules. A small on-device model reads what you write to spot patterns the rules can't reach. No internet. No AI chat. No subscriptions.

Layer 1 — Your rules Layer 2 — On-device pattern model

Layer 1 — Your keyword rules

You write the rules. JotFolio matches words you said to watch for.

  • If you write "lasagna" → label it #recipes
  • If you write "grandma" → suggest linking to [[Grandma's recipes]]
  • Plain word matching, exact, predictable
  • Edit rules anytime in plain text

Layer 2 — Pattern model (MiniLM)

A small file that reads notes and learns what they're about, without you defining anything.

  • Search by meaning, not just words
  • Find similar notes you didn't link
  • Suggest tags for new notes
  • Group notes into themes automatically
  • Catch when you've written the same thing twice

How it works (in plain English)

Curious what's actually happening? Here's the pipeline. Nothing leaves your computer at any step.

📝

Plain text on your disk, exactly as you wrote it
→
🔢
MiniLM reads it
Turns the words into a list of 384 numbers ("fingerprint")
→
💾
Saved next to the note
Numbers stored locally — never sent anywhere
→
🔍
Used for search + suggestions
Compare fingerprints to find similar notes

↑ MiniLM ships inside JotFolio (~50MB file, one-time). Like a calculator that came with the app. Never phones home.

Why both layers, instead of just one?

Rules alone are predictable but rigid. They only fire on words you defined. A new note about a new topic gets nothing.

The pattern model alone is flexible but opaque. You can't tell why it suggested a tag, and there's no way to say "no, this word always means X."

Together: you keep control via rules. The model fills in the gaps the rules don't reach. When the model suggests something, the rule editor offers to turn it into a permanent rule — so the model's pattern recognition becomes your explicit, editable taxonomy over time.

Charter check: ✓ Local-first (model runs on your machine). ✓ Plain-text durability (rules + notes stay readable in any text editor). ✓ No external content shipping (note contents never leave the app). ✓ Pure-function indexes (fingerprints rebuild from files).
